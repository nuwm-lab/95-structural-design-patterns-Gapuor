# 📋 Звіт про виконану лабораторну роботу

## Тема: Структурні патерни проектування
## Завдання: Реалізувати адаптер для різних типів залізничних колій (європейські та ширококолійні)

---

## ✅ Завершені етапи

### 1. Аналіз завдання
- Обрано варіант №9 з примітками про адаптер залізничних колій
- Визначені два типи колій:
  - Європейська (UIC стандарт): 1435 мм
  - Ширококолійна (Russian standard): 1676 мм
  - Різниця: 241 мм

### 2. Реалізація паттерну Adapter
Створено два адаптери:
- `EuropeanToWideGaugeAdapter` - конвертує європейську колію в ширококолійну
- `WideToEuropeanGaugeAdapter` - конвертує ширококолійну колію в європейську

### 3. Застосування Facade паттерну
- Клас `RailwayAdapterFacade` спрощує роботу з адаптерами
- Фасад повертає структуровані дані (не виводить сам)
- Підтримує отримання оригінальних та адаптованих об'єктів

### 4. Централізація константи конверсії
- Створений клас `GaugeConverter` зі статичною логікою
- Константа `GaugeConversionDifference = 241` мм
- Обидва адаптери використовують це значення (дотримання DRY)

### 5. Розділення коду по файлам
Структура адаптерів:
```
Adapters/
├── GaugeConverter.cs (логіка конверсії)
├── EuropeanToWideGaugeAdapter.cs
├── WideToEuropeanGaugeAdapter.cs
└── RailwayAdapterFacade.cs (фасад + структури)
```

### 6. Юніт-тестування
- Клас `RailwayAdapterTests` з комплексним набором тестів
- 20 тестів охоплюють:
  - Коректність конверсії ширини
  - Адаптацію стандартів та описань
  - Централізовану логіку конверсії
  - Роботу фасаду
  - Валідацію параметрів та граничні випадки

**Результат:** ✅ **20/20 тестів пройдено**

### 7. Дотримання Code Conventions
- ✅ PascalCase для публічних членів класу
- ✅ camelCase для приватних полів та локальних змінних
- ✅ XML документація для всіх публічних методів
- ✅ Дотримання Microsoft C# Coding Conventions

### 8. Дотримання принципу інкапсуляції
- ✅ Приватні поля (`private readonly`) захищені від зовнішнього доступу
- ✅ Публічні методи надають контрольований доступ до даних
- ✅ Валідація параметрів (null-checks) з ArgumentNullException
- ✅ Фасад повертає копії даних, а не прямо змінює внутрішній стан

## 🎯 Реалізовані паттерни проектування

### 1. **Adapter Pattern**
```
┌─────────────────┐    Adapter    ┌──────────────────┐
│  European Rail  │ ←────────────► │ Wide Gauge Rail  │
│   (1435 mm)     │   (241 mm)     │    (1676 mm)     │
└─────────────────┘                └──────────────────┘
```

### 2. **Facade Pattern**
```
┌──────────────────────────────────┐
│   RailwayAdapterFacade           │
│  • GetEuropeanRailway()          │
│  • GetWideGaugeRailway()         │
│  • GetEuropeanAsWideGauge()      │
│  • GetWideGaugeAsEuropean()      │
│  • GetComparison()               │
└──────────────────────────────────┘
```

## 💡 Ключові особливості реалізації

1. **Інкапсуляція:** Всі поля приватні, доступ надається через публічні методи
